# Android — Expo prebuild workflow

The `android/` directory should be generated by Expo, not by `react-native init`.

## Generate or refresh the native project

```bash
cd packages/mobile-shell
npx expo prebuild --platform android
```

If native dependencies or config have drifted, regenerate from a clean slate:

```bash
cd packages/mobile-shell
npx expo prebuild --clean --platform android
```

## Local development
1. Run `cd packages/mobile-shell && npx expo run:android` to create a local dev build.
2. Open the generated `android/` project in Android Studio only when you need native debugging.
3. When native module updates are not picked up, restart Metro with `npx expo start -c` and reinstall the dev build.
