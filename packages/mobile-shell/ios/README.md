# iOS — Expo prebuild workflow

The `ios/` directory should be generated by Expo, not by `react-native init`.

## Generate or refresh the native project

```bash
cd packages/mobile-shell
npx expo prebuild --platform ios
```

If native dependencies or config have drifted, regenerate from a clean slate:

```bash
cd packages/mobile-shell
npx expo prebuild --clean --platform ios
```

## Local development
1. Run `cd packages/mobile-shell && npx expo run:ios` to create a local dev build.
2. If you need to inspect native settings manually, open the generated `.xcworkspace` in Xcode.
3. When WebView or other native modules behave strangely, restart Metro with `npx expo start -c` and reinstall the dev build or Expo Go client.
